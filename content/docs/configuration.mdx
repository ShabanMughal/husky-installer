# Configuration

All configuration files are automatically created by Husky Installer. You can customize them to fit your project's needs.

## Husky Configuration

Husky hooks are stored in the `.husky` directory. Each hook is a shell script that runs at specific git events.

## Prettier Configuration

Prettier configuration is stored in `.prettierrc`. Here's a recommended setup:

<DocsCodeBlock
  lang="json"
  code={`{
  "semi": true,
  "trailingComma": "all",
  "singleQuote": true,
  "printWidth": 80,
  "tabWidth": 2,
  "arrowParens": "always",
  "endOfLine": "lf"
}`}
/>

### Customizing Prettier

You can modify `.prettierrc` to match your team's style preferences. See the [Prettier documentation](https://prettier.io/docs/en/configuration.html) for all options.

## ESLint Configuration

ESLint configuration is in `eslint.config.js`. Example configuration:

<DocsCodeBlock
  lang="javascript"
  code={`export default [
  {
    rules: {
      'no-console': 'warn',
      'no-unused-vars': 'error',
      'prefer-const': 'error',
      'no-var': 'error',
    },
  },
];`}
/>

### Adding ESLint Rules

Extend the configuration with your preferred rules or use a preset:

<DocsCodeBlock
  lang="javascript"
  code={`import js from '@eslint/js';

export default [
  js.configs.recommended,
  {
    rules: {
      // Your custom rules
    },
  },
];`}
/>

## Commit Message Formats

Husky Installer supports two commit message formats:

### Conventional Commits

Follows the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
type(scope): subject

body

footer
```

## Customizing Hooks

You can modify hook files in `.husky/` to add custom behavior:

<DocsCodeBlock
  lang="bash"
  code={`# .husky/pre-commit
#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

# Run your custom scripts

npm run custom-check
npm run lint
npm run format`}
/>

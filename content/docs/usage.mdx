# Usage

Husky Installer offers two ways to configure your git hooks: **CLI mode** and **Web interface**.

## CLI Mode

Run the interactive CLI in your project directory:

<PackageManagerTabs
  commands={{
    npm: 'npx husky-installer',
    pnpm: 'pnpm dlx husky-installer',
    yarn: 'yarn dlx husky-installer',
    bun: 'bunx husky-installer',
  }}
/>

The CLI will guide you through:

<Steps>
  <Step>
    ### Package Manager Detection Automatically detects npm, yarn, pnpm, or bun
    in your project.
  </Step>
  <Step>
    ### Prettier Setup Choose whether to add code formatting with Prettier for
    consistent code style.
  </Step>
  <Step>
    ### ESLint Configuration Optionally add linting rules to catch errors and
    enforce code quality standards.
  </Step>
  <Step>
    ### Commit Message Format Select emoji or conventional commit format for
    your commit messages.
  </Step>
  <Step>
    ### Husky Installation Automatically installs and configures Husky with all
    selected options.
  </Step>
</Steps>

## Web Interface

Start the web interface for visual configuration:

```bash
npm run dev
```

Then open your browser to:

- Configure hooks visually
- Preview generated scripts
- Copy configuration commands

## Available Hooks

Husky Installer sets up two main hooks:

<Accordions type="single">
  <Accordion title="Pre-commit Hook">
    Runs **before each commit** to ensure code quality:

    - ‚úÖ **Format code** with Prettier
    - ‚úÖ **Lint code** with ESLint
    - ‚úÖ **Run tests** (optional, if configured)

    If any of these fail, the commit is blocked until issues are resolved.

  </Accordion>

  <Accordion title="Commit Message Hook">
    Validates commit messages to ensure they follow your chosen format:

    - üìù **Conventional Commits** - `feat:`, `fix:`, `docs:`, etc.
    - üé® **Emoji Commits** - Automatically adds emoji prefixes
    - ‚ú® **Custom Patterns** - Enforce your own commit message rules

    This ensures consistent commit history across your project.

  </Accordion>
</Accordions>
